BuildTool documentation

BuildTool is a command line utility that processes XAML files into a format that is optimized for runtime loading. At this point, XAML files are converted to binary format and images can be compressed into a more efficient format. Once processed, files will have the .nsb extension and are only compatible with noesisGUI.

Usage

BuildTool is located in the /Bin directory of the SDK and is expected to be used from the command line. Both single filenames and recursive folders can be imported. To see the full list of command line options and usage instructions, execute BuildTool without any additional arguments and help output will be printed.

> Resource.BuildTool.exe

------------------------------------------------------------------------------
 Build Tool - noesisGUI Importer v0.91
------------------------------------------------------------------------------

 Usage:

     BuildTool filename  [options]
     BuildTool directory [options]

 Options:

     --platform platform
         Target platform used to build input resources

     --outdir directory
         Output folder where imported resources will be copied

     --tempdir directory
         Temporary folder for intermediate files

     --modstamp
         Do not run if target file is older than source

     --header
         Generates a c-header file instead of a binary one

     --deps
         In this mode only XAML files and their dependencies,
         like images and fonts, are processed

     --verbose
         Increases verbosity

     --fatalerrors
         Causes the importer to abort on the first error occurred


 Registered platforms:

     DX9
     DX11
     GL
     GLES
     iOS
     Android

Loading Resources

At execution time noesisGUI needs to know where the preprocessed resources generated by the BuildTool are located. For example, in case your resources are located inside a disk folder you can use the following code:

// Look for resources inside "MyData" folder located in the current working directory
Noesis::GUI::AddResourceProvider("MyData");

For more advanced scenarios we provide the class ResourceProvider, a base class with several virtual functions that can be reimplemented to customize the way resources are loaded. For example, if you used the -header switch with the BuildTool, you can provide a memory stream this way:

namespace
{
// This header was automatically generated by BuildTool using the -header switch
#include "Button.xaml.h"
}

class MyProvider: public ResourceProvider
{
private:
    Ptr<Core::IStream> RequestFile(const NsChar* name)
    {
        if (String::Compare(name, "Button.xaml", IgnoreCase_True) == 0)
        {
            return *new MemoryStreamR(Button_xaml, sizeof(Button_xaml));
        }
    }
};

If more control is needed you can implement your own stream by deriving from BaseStream.
